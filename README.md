# co_editor
# 技术栈
- 前端采用svelte + yjs
- 后端使用了phoenix
- 数据库暂时使用了postgresql

# 前端页面
<img src="./img/frontend_page.png">，demo中设计了三个页面：
- 登陆页面（注册功能不是主要的，暂时在db中写死两个用户）
- 文档的管理页面，有三个功能：
  - 右上角点击之后能够创建一个新的空白文档，
  - 暂时不引入文档所有权的问题，只要是这个系统中有的文档就任何用户都可以写入，编辑，删除
  - 当点开了一个文档后，会将该文档的内容显示在页面上

用户编辑文档页面的操作：
- 字符输入操作，特指键入的字符
- 用户采用`/table, /img`可以导入文本，图片（当然可以拓展到audio，href数据）
